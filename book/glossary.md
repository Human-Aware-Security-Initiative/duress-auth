| Термін                    | Визначення                                                                                                                                   |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **Duress Authentication** | Автентифікація, що враховує можливість примусу.                                                                                              |
| **Duress PIN**            | Спеціальний PIN, який зовні виглядає як звичайний вхід, але активує визначену політику безпеки.                                              |
| **Silent Alert**          | Непомітне для нападника повідомлення про можливий примус.                                                                                    |
| **Safe Mode**             | Обмежений режим роботи застосунку без доступу до критичних функцій.                                                                          |
| **Forced Session**        | Сесія, яка може бути створена під примусом і тому має знижений рівень довіри.                                                                |
| **Trust Level**           | Поточна оцінка довіри до автентифікованої сесії.                                                                                             |
| **Risk Escalation**       | Підвищення рівня реагування системи при виявленні ознак ризику.                                                                              |
| **Guardian Contact**      | Довірена особа або сервіс, який отримує сповіщення у визначених сценаріях.                                                                   |
| **Human Integrity**       | Припущення, що користувач є вільним і самостійно приймає рішення. Якщо це припущення не виконується, система повинна змінити свою поведінку. |
